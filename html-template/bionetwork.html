Overview

Lichen is a new project at ISB that aims to provide network and heat map visualization in web browsers. It provides components that can be used as interactive figures in any website. It is also being adapted for use from R and the Gaggle.

Lichen's network visualizer includes several novel features. The CircularHeatmap node render uses concentric circles to display time series data on nodes. New layouts algorithms have been designed to better reveal clusters and patterns in networks.

The entire Lichen code base is implemented in actionscript/flash and platform-independent javascript to ensure that visualizations can be viewed in almost any web browser and operating system. It complies with the Google Visualization API which provides a standardized way for web developers to load data and listen for events. It is purely open source and and can be modified and redistributed freely.

By: rbressler, sdee

Loading

The script requires the google jsapi and the Prototype Javascript library (1.6 and above). To load the script place bionetwork.js and bionetwork.swf in the same directory as your page and use this code: 

    <script type="text/javascript" src="http://www.google.com/jsapi"></script>
    <script type="text/javascript">
      google.load("visualization", "1", {});
      google.load("prototype", "1.6");
    </script>    
    
    <script type="text/javascript" src="bionetwork.js"></script>

If you wish to load bionetwork.js and bionetwork.swf from another location (/js/bionetwork.js) you will need to open the bionetwork.js in a text editor and find-and-replace bionetwork.swf to the appropriate path.

The visualization's javascript class name is org.systemsbiology.visualization.BioNetwork 

    var network = new org.systemsbiology.visualization.BioNetwork(document.getElementById('networkContainer'));

Data Format

This data parameter should be a data table adjacency list with the following columns in positions 0-2:

Position Title Description
0 edge_id A string specifying an id to be associated with the edge. Can be null or "".
1 interactor_value1 A string specifying the first interactor. Used as the from interactor if the edge is directed.
2 interactor_value2 A string specifying the second interactor. Used as the to interactor if the edge is directed.

And the following optional columns specified by title:

Title Description
color A string specifying the color of the edge in the form "0xAARRGGBB" representing alpha, red, green and blue hex values.
directed A boolean specifying weather the edge is directed or not. Edges are assumed to be undirected unless TRUE is provided in this column.
type A string specifying what type of arrow to use for a directed edge. Can be "TRIANGLE", "LINES" or "NONE"; the default is "TRIANGLE."
sources A string containing a comma separated list of for use of edge types for use in the multi edge renderer.

Configuration Options

Basic Options:

Name Type Default Description
width Number 630 The width of the visualization in pixels.
height Number 630 The height of the visualization in pixels.
padding Number 5 The amount of padding between the edge of the visualization and the edge of the swf object.
layout String "Circular" The type of layout to use. Can be "Circular", "ForceDirected", "extendedForceDirected", "3dSVD", "GoogleDataTableDriven" or "radialTree."
center String The node first node listed in data. A string specifying which node is the center or root node used by some layouts.
node_fillColor Color String "0x880055cc" The default node color specified in hex as "0xAARRGGBB."
node_lineWidth Number
node_tooltips Bool FALSE
node_labels_radial Bool FALSE
node_clickdrag Bool TRUE Can nodes be dragged around.
edge_lineWidth Number 2 The default width of each edge.
edge_lineColor Color String "0x66000000" The defaulte edge color.
selection_display String Default is to display nodes and edge selection "nodes","edges" or "none". To view edge selection you may need to modify the other selection display options.
selection_persistDisplay bool true Should seletion dislapy be perminant or flash.
selection_lineColor Color String If present selection display will use this color.
selection_lineWdith Number 3 Used to resize lines/boarders to display selection.
selection_lineAlpha Number 1 Used to change the alpha of lines/boarders to display selection.

Advanced Options:

Name Type Default Description
legend Array of Objects An array of objects used to create a legend. Example : [ {color: 0x3366CC, size: 0.75, label: "HPRD"}, {color: 0x339900, size: 0.75, label: "MINT"," shape:"TRIANGLE_RIGHT"} ]
node_renderer String Used to specify custom node renders, can be set too "CircularHeatmap."
edge_renderer String Used to specify custom edge renders, can be set to "multiedge."
edge_router String Used to specify custom edge routers. Can be set to 'bundled'.
CircularHeatmap_maxval Number The maximum value for use in the circular heatmap.
CircularHeatmap_minval Number The minumum value for use in the circular heatmap.
nodeClusterPositions Array NULL An optional array specifying positions at which to fix node clusters. Only used by the extended force directed layout. Should be of the format [{x:postion1x,y:position1y},{x:postion1x,y:position1y},...].

Optional DataTables:

Name Description
layout_data Required Columns:
Position Description
0 Node Id
1-n Columns 1-n are used to specify arbitrary data. Columns with titles begining with "t_" will be views as time points to be used by the circular heatmap if it is enabled.
Optional Columns Specified by Title:
Title Description
shape A string specifying one of the flare shape constants: "CIRCLE", "SQUARE", "DIAMOND", "TRIANGLEDOWN", "TRIANGLEUP", "TRIANGLERIGHT", "TRIANGLELEFT", "CROSS" or "X."
color A string of the form "0xAARRGGBB" specifying a color.
size A number specifying a size multiplier to apply to the node.
x An int specifying the x position of the node in pixels.
y An int specifying the y position of the node in pixels.
node_data Used for specifying a list of data to be used with the circularheatmap, needs to be cleaned up and may be merged with layout data. Columns:
Position Description
0 Node Id
1-n Columns 1-n are used to specify arbitrary data. Columns with titles beginning with "t_" will be views as time points to be used by the circular heatmap if it is enabled.
nodeClusters Used by cluster aware layouts to determine which clusters nodes belong to. Column 0 should contain node ids and column 1-n should be boolean columns representing cluster membership. Cluster names are determined from column labels.

Methods

Method Return Type Description
draw(data, options) none Draws the heatmap.
getSelection() Array of selected edges/rows Standard getSelection implementation.
getSelectionNodes() Array of selected nodes Standard getSelection implementation. Selection elements are cells when a heatmap cell is clicked, rows when a row label is clicked and columns when a column label is clicked.
setSelection(selection) none selection is an array of selected items. Items can be either nodes specified as {node;"node_id"} or edges specified by row number in the data parameter: {row:row_num}.
add_edge_from_to none beta
remove_node_by_name none beta
remove_edge_from_to none beta

Events

Name Description Properties
select Standard select event None
nodeclick Fired on a node click, handler should accept a single parameter, target. target.node.name
nodedoubleclick Fired on a node double click, handler should accept a single parameter, target. target.node.name
edgeclick Fired on edge click, handler should accept a single parameter but the edge id is not currently available from this event.

Data Policy

All code and data are processed and rendered in the browser. No data is sent to any server.
